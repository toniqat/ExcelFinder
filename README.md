# 엑셀 파일 검색 도구 (Excel Finder)

엑셀 파일 내의 텍스트를 빠르게 검색할 수 있는 도구입니다. 여러 엑셀 파일에서 특정 값을 찾고, 결과를 쉽게 확인할 수 있습니다.

## 주요 기능

- 여러 엑셀 파일(.xlsx, .xls, .xlsm)에서 동시에 검색
- 정확한 값 일치 또는 부분 일치 검색 지원
- 병렬 처리를 통한 빠른 검색 속도
- 검색 결과 클릭 시 해당 행의 데이터 표시
- 대용량 엑셀 파일도 빠르게 처리 (캐싱 기능)
- 'Unused' 헤더를 가진 열에서 값이 1인 행은 검색에서 제외
- **스마트 로딩 화면**: 애플리케이션 시작 시 진행 상황을 실시간으로 표시

## 사용 방법

1. **파일 추가**: '파일 추가' 또는 '폴더 추가' 버튼을 클릭하여 검색할 엑셀 파일을 추가합니다.
2. **검색 옵션 설정**: 
   - 검색할 값을 입력합니다.
   - '값 일치' 또는 '값 포함' 옵션을 선택합니다.
   - 병렬 처리 수를 조정합니다 (기본값은 최대 4개, CPU 코어 수를 넘지 않음).
3. **검색 실행**: '검색' 버튼을 클릭하여 검색을 시작합니다.
4. **결과 확인**: 검색 결과가 표시되면 항목을 더블 클릭하여 해당 행의 데이터를 확인할 수 있습니다.

## 실행 파일 빌드 방법

### 필요 조건

- Python 3.6 이상
- 필요한 패키지들 (자동으로 설치됨):
  - pandas
  - numpy
  - PyQt5
  - xlrd
  - openpyxl
  - pyinstaller

### 빌드 방법

1. 필요한 패키지 설치 (선택 사항, build_exe.py에서 자동으로 설치됨):

```
pip install -r requirements.txt
```

2. 터미널에서 다음 명령어를 실행합니다:

```
# 단일 실행 파일로 빌드 (배포 용이, 시작 속도 느림)
python build_exe.py --type onefile

# 또는 디렉토리로 빌드 (빠른 시작 속도, 배포 복잡)
python build_exe.py --type onedir
```

3. 빌드가 완료되면 `dist` 폴더에 결과물이 생성됩니다:
   - `onefile` 모드: `ExcelFinder.exe` 파일 하나만 생성됨
   - `onedir` 모드: `ExcelFinder` 폴더와 그 안에 여러 파일이 생성됨

4. 이 파일(들)을 다른 컴퓨터에 복사하여 실행할 수 있습니다 (Python 설치 불필요).

### 빌드 유형 선택 가이드

- **onefile 모드**: 단일 실행 파일로 빌드합니다.
  - 장점: 배포가 간편합니다. 하나의 .exe 파일만 복사하면 됩니다.
  - 단점: 시작 시간이 느립니다. 실행 시 임시 폴더에 모든 파일을 압축 해제해야 합니다.

- **onedir 모드**: 폴더와 여러 파일로 빌드합니다.
  - 장점: 시작 시간이 빠릅니다. 압축 해제 과정이 없습니다.
  - 단점: 배포 시 전체 폴더를 복사해야 합니다.

### 수동 빌드 방법

PyInstaller를 직접 사용하여 빌드하려면:

1. 필요한 패키지 설치:

```
pip install -r requirements.txt
```

2. PyInstaller 설치 (requirements.txt에 포함되어 있음):

```
pip install pyinstaller
```

3. 빌드 명령어 실행:

```
# 단일 실행 파일로 빌드
pyinstaller --name=ExcelFinder --onefile --windowed --noupx --clean --noconfirm main.py

# 또는 디렉토리로 빌드
pyinstaller --name=ExcelFinder --onedir --windowed --noupx --clean --noconfirm main.py
```

## 프로젝트 구조

- `main.py`: 애플리케이션 진입점
- `config.py`: 설정 및 공통 임포트
- `file_processor.py`: 엑셀 파일 처리 기능
- `search_worker.py`: 병렬 검색 작업자
- `sheet_viewer.py`: 시트 뷰어 다이얼로그
- `main_app.py`: 메인 애플리케이션 창
- `build_exe.py`: 실행 파일 빌드 스크립트

## 시작 속도 최적화

애플리케이션 시작 속도가 느린 경우 다음 방법을 시도해 볼 수 있습니다:

1. **디렉토리 모드로 빌드**: `--type onedir` 옵션을 사용하여 빌드하면 시작 속도가 크게 향상됩니다.
   ```
   python build_exe.py --type onedir
   ```

2. **스플래시 화면**: 애플리케이션이 로딩 중임을 표시하는 스플래시 화면이 구현되어 있습니다.

3. **자세한 최적화 방법**: 더 자세한 최적화 방법은 `startup_optimization.md` 파일을 참조하세요.

## 주의사항

- 매우 큰 엑셀 파일(수백 MB 이상)을 처리할 때는 충분한 메모리가 필요할 수 있습니다.
- 복잡한 수식이나 매크로가 포함된 엑셀 파일은 처리 시간이 더 오래 걸릴 수 있습니다.
- 빌드된 실행 파일은 Windows 환경에서만 실행됩니다.
- 단일 실행 파일(.exe)로 빌드한 경우 첫 실행 시 시간이 오래 걸릴 수 있습니다.

## 문제 해결

### 엑셀 파일 열기 오류

일부 엑셀 파일(.xls)을 읽을 때 '파일이름 열기 실패: 추가 오류:' 메시지가 나타날 경우 다음 방법을 시도해 보세요:

1. **최신 버전 사용**: 최신 버전의 프로그램을 사용하고 있는지 확인하세요. 최신 버전에는 다양한 엑셀 파일 형식을 처리하기 위한 개선 사항이 포함되어 있습니다.

2. **파일 형식 확인**: 문제가 발생하는 파일이 실제로 .xls 형식인지 확인하세요. 파일 확장자가 .xls로 되어 있어도 실제로는 다른 형식일 수 있습니다.

3. **파일 복구 시도**: 
   - Excel에서 파일을 열고 다시 저장해 보세요.
   - 또는 Excel의 '파일 > 열기 > 찾아보기'에서 '모든 Excel 파일'로 설정한 후 '열기' 대신 아래쪽 화살표를 클릭하고 '복구하여 열기'를 선택하세요.

4. **인코딩 문제**: 한글이 포함된 파일의 경우 인코딩 문제가 발생할 수 있습니다. 이 경우 Excel에서 파일을 열고 다른 이름으로 저장(다른 형식 선택)하여 시도해 보세요.

5. **pyxlsb 설치**: 일부 바이너리 Excel 파일(.xlsb)을 처리하기 위해 pyxlsb 라이브러리가 필요할 수 있습니다. 다음 명령어로 설치할 수 있습니다:
   ```
   pip install pyxlsb
   ```

6. **손상된 파일**: 파일이 손상되었을 수 있습니다. Excel의 복구 기능을 사용하거나, 파일을 다시 만들어야 할 수도 있습니다.

7. **메모리 부족**: 매우 큰 파일을 처리할 때 메모리 부족 오류가 발생할 수 있습니다. 이 경우 다른 프로그램을 종료하거나 컴퓨터를 재시작한 후 다시 시도해 보세요.

8. **오류 로그 확인**: 오류 로그 영역에 표시되는 자세한 오류 메시지를 확인하여 문제의 원인을 파악할 수 있습니다.

### Microsoft Excel 97-2003 워크시트(.xls) 파일 처리

Excel 97-2003 형식(.xls)의 파일은 최신 Excel 형식(.xlsx)과 다른 방식으로 처리됩니다. 이러한 파일을 읽을 때 문제가 발생하는 경우 다음 방법을 시도해 보세요:

1. **xlrd 라이브러리 사용**: 이 프로그램은 .xls 파일을 읽기 위해 xlrd 라이브러리를 사용합니다. 최신 버전의 xlrd는 .xls 파일만 지원하며 .xlsx 파일은 지원하지 않습니다.

2. **특별 처리 옵션**:
   - **인코딩 설정**: 한글이 포함된 .xls 파일의 경우 `encoding_override='cp949'` 옵션을 사용하여 처리합니다.
   - **불규칙한 행 허용**: 행마다 열 수가 다른 경우 `ragged_rows=True` 옵션을 사용합니다.
   - **메모리 매핑 비활성화**: 오래된 .xls 파일의 경우 `xlrd.mmap_mode = False` 설정이 도움이 될 수 있습니다.

3. **파일 변환 방법**:
   - Excel에서 파일을 열고 '다른 이름으로 저장' > 'Excel 통합 문서(.xlsx)'를 선택하여 최신 형식으로 변환할 수 있습니다.
   - 또는 '다른 이름으로 저장' > 'Excel 97-2003 통합 문서(.xls)'를 선택하여 같은 형식으로 다시 저장하면 파일이 정리되어 읽기 쉬워질 수 있습니다.

4. **대체 방법**:
   - 파일을 CSV 형식으로 저장한 후 다시 시도해 보세요.
   - LibreOffice Calc와 같은 다른 스프레드시트 프로그램으로 파일을 열고 다시 저장해 보세요.

5. **Excel 97-2003 파일 특성**:
   - 최대 65,536행 x 256열의 제한이 있습니다.
   - 복잡한 서식, 매크로, 차트 등이 포함된 경우 읽기 어려울 수 있습니다.
   - 일부 기능은 최신 Excel 형식에서만 지원됩니다.

### 상세 오류 로그 확인

엑셀 파일을 열 때 발생하는 오류를 더 자세히 분석하려면 다음 기능을 활용하세요:

1. **오류 로그 영역**: 
   - 프로그램 메인 화면의 왼쪽 하단에 있는 '오류 로그' 영역에서 상세한 오류 정보를 확인할 수 있습니다.
   - 오류 메시지는 색상으로 구분되어 가독성이 향상되었습니다.
   - '오류 위치' 항목을 클릭하면 상세한 스택 트레이스를 확인할 수 있습니다.

2. **로그 파일**:
   - 모든 오류는 자동으로 로그 파일에 기록됩니다.
   - 로그 파일은 프로그램 설치 폴더의 `logs/excel_errors.log` 파일에 저장됩니다.
   - 이 파일에는 오류 발생 시간, 파일 경로, 상세 오류 메시지 등이 포함됩니다.

3. **오류 정보 분석**:
   - 파일 크기, 파일 확장자 등의 기본 정보
   - 각 단계별 시도와 실패 원인 (오류 1, 오류 2, 오류 3 등)
   - 오류 유형과 내용
   - 오류 발생 위치 (스택 트레이스)

이 정보를 활용하면 특정 엑셀 파일을 열 수 없는 정확한 원인을 파악하고 적절한 해결책을 찾는 데 도움이 됩니다.
