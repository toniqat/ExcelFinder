# 검색 예외 처리 기능 가이드

## 🎯 기능 개요

ExcelFinder의 검색 예외 처리 기능을 사용하면 특정 조건에 따라 검색 결과에서 컬럼이나 행을 제외할 수 있습니다.

## 🔧 두 가지 예외 처리 방식

### 1. **컬럼 검색 제외** (왼쪽 패널)
- **기능**: 지정한 이름의 컬럼은 검색 대상에서 완전히 제외
- **용도**: 개인정보, 시스템 컬럼 등 검색하지 않아야 할 컬럼 지정
- **예시**: "주민번호", "비밀번호", "내부코드" 등

### 2. **조건부 행 제외** (오른쪽 패널)  
- **기능**: 지정한 컬럼에 값이 있는 행은 검색 결과에서 제외
- **용도**: 삭제된 데이터, 비활성 데이터 등 제외
- **예시**: "삭제여부", "상태", "활성화" 등

## 📝 설정 방법

### 1. 검색 예외 처리 다이얼로그 열기
- 메인 화면에서 **"설정"** 버튼 클릭
- 또는 검색 옵션 영역의 **"검색 예외 처리 설정"** 버튼 클릭

### 2. 컬럼 검색 제외 설정 (왼쪽)
1. **"추가"** 버튼 클릭
2. **헤더 이름** 입력 (정확한 컬럼명)
3. **메모** 입력 (선택사항, 설명 용도)
4. **확인** 클릭

### 3. 조건부 행 제외 설정 (오른쪽)
1. **"추가"** 버튼 클릭  
2. **헤더 이름** 입력 (조건 확인할 컬럼명)
3. **메모** 입력 (선택사항)
4. **확인** 클릭

## 💡 실제 동작 예시

### 예시 데이터
```
| 번호 | 이름   | 나이 | 이메일        | 삭제여부 | 상태   |
|------|--------|------|---------------|----------|--------|
| 1    | 홍길동 | 30   | hong@test.com |          | 활성   |
| 2    | 김철수 | 25   | kim@test.com  | Y        | 비활성 |
| 3    | 이영희 | 28   | lee@test.com  |          | 활성   |
```

### 설정 예시
- **컬럼 검색 제외**: "번호", "나이" 
- **조건부 행 제외**: "삭제여부"

### 검색 결과 ("홍길동" 검색 시)
- ✅ 1행의 "이름" 컬럼에서 "홍길동" 발견 → **검색 결과에 포함**
- ❌ 1행의 "번호" 컬럼은 검색 제외 → 검색 안함
- ❌ 2행은 "삭제여부"에 "Y" 값 있음 → **행 전체 제외**

## 🔍 디버그 모드

검색 예외 처리가 올바르게 작동하는지 확인하려면:

### 1. 디버그 모드 실행
```bash
# 배치 파일 사용
debug_mode.bat

# 또는 직접 실행
set EXCEL_FINDER_DEBUG=1
python main.py
```

### 2. 디버그 출력 예시
```
[DEBUG] CSV 파일 data.csv 제외 처리 정보:
  원본 제외 헤더: ['번호', '나이 (메모: 사용자 나이)']
  정리된 제외 헤더: ['번호', '나이']
  제외될 컬럼: ['번호', '나이']
  값 있으면 제외될 컬럼: ['삭제여부']

    [DEBUG] 행 2 스킵 (값 있는 컬럼: ['삭제여부'])
    [DEBUG] 컬럼 '번호' 제외 (검색 스킵)
```

## 🛠 고급 기능

### 1. 중복 헤더 처리
- Excel에서 중복 헤더는 자동으로 ".1", ".2" 접미사 추가
- ExcelFinder는 이를 자동으로 처리하여 원본 헤더명으로 매칭

**예시**:
- 실제 헤더: `["이름", "나이", "이름.1", "주소"]`
- 설정에서 "이름" 제외 → `["이름", "이름.1"]` 모두 제외

### 2. 메모 기능
- 각 설정에 설명을 추가할 수 있음
- 메모는 설정 저장 시 포함되지만 실제 검색에는 영향 없음
- 팀원들과 설정을 공유할 때 유용

### 3. 편집 및 삭제
- **편집**: 설정 항목 선택 후 "편집" 버튼
- **삭제**: 설정 항목 선택 후 "선택 항목 삭제" 버튼
- 다중 선택으로 여러 항목 한번에 삭제 가능

## ⚠️ 주의사항

### 1. 정확한 헤더명 입력
- 헤더명은 **대소문자 구분** 및 **공백 포함** 정확히 입력
- 오타가 있으면 예외 처리가 작동하지 않음

### 2. 검색 예외 처리 적용 확인
- 메인 화면에서 **"검색 예외 처리 적용"** 체크박스 확인
- 체크 해제 시 모든 예외 처리 무시됨

### 3. 성능 고려사항  
- 예외 처리 규칙이 많을수록 검색 속도가 약간 느려질 수 있음
- 하지만 불필요한 데이터를 제외하므로 전체적으로는 효율적

## 🚀 활용 팁

### 1. 자주 사용하는 패턴
```
컬럼 검색 제외:
- "ID", "번호", "코드" (시스템 식별자)
- "생성일", "수정일" (타임스탬프)  
- "비밀번호", "주민번호" (민감정보)

조건부 행 제외:
- "삭제여부" (Y/N 플래그)
- "상태" (비활성/삭제 등)
- "테스트여부" (테스트 데이터 구분)
```

### 2. 팀 단위 설정 공유
- `config/app_settings.json` 파일을 팀원들과 공유
- 표준화된 예외 처리 규칙 적용 가능

### 3. 단계별 적용
1. 먼저 디버그 모드로 동작 확인
2. 소량 데이터로 테스트
3. 대량 데이터에 적용

이제 검색 예외 처리 기능이 완벽하게 작동하며, 메모가 포함된 설정도 올바르게 처리됩니다! 🎉
